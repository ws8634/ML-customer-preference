# 架构设计文档

## 1. 架构师决策记录 (ADR)

### 决策1: 采用模块化三层架构

**问题**: 如何组织代码结构以支持可维护性和可扩展性？

**决策**: 采用三层架构设计，将系统拆分为三个独立模块：
- 训练层 (model_training.py)
- 评估层 (model_evaluation.py)
- 服务层 (api_service.py)

**理由**:
- 遵循关注点分离原则
- 各模块可以独立开发、测试和部署
- 便于未来扩展（例如更换模型算法、添加新的评估指标）
- 支持团队协作，不同人员可以负责不同模块

**替代方案**:
- 单一文件架构：简单但难以维护
- 微服务架构：过度复杂，超出项目需求

### 决策2: 使用投票集成模型

**问题**: 如何提高模型性能和鲁棒性？

**决策**: 使用软投票（VotingClassifier）集成随机森林和逻辑回归两种算法。

**理由**:
- 集成学习通常比单一模型性能更好
- 随机森林擅长处理非线性关系，逻辑回归提供线性基准
- 软投票考虑概率分布，比硬投票更稳定
- 符合技术栈限制（不使用XGBoost等）

**替代方案**:
- 单一决策树：性能较差
- 更多基分类器：增加复杂度

### 决策3: 采用REST API接口

**问题**: 如何提供标准化的外部调用接口？

**决策**: 使用Flask框架提供RESTful API服务。

**理由**:
- REST是业界标准，易于被外部系统集成
- HTTP协议通用性强，支持各种客户端
- Flask轻量级，适合小型项目
- 支持JSON数据交换格式

**替代方案**:
- gRPC：性能更好但复杂度更高
- 命令行接口：不适合远程调用

## 3. 模块详细设计

### 3.1 训练层 (model_training.py)

**核心职责**:
- 数据加载和预处理
- 模型构建和训练
- 模型持久化

**关键接口**:
- `load_data(file_path)`: 加载数据集
- `preprocess_data(X)`: 数据标准化
- `train_ensemble_model(X, y)`: 训练集成模型

**数据流向**:
```
CSV文件 → DataFrame → 标准化 → 模型训练 → 保存模型文件
```

### 3.2 评估层 (model_evaluation.py)

**核心职责**:
- 模型性能评估
- 评估指标计算
- 结果展示

**关键接口**:
- `evaluate_model(model, X_test, y_test)`: 计算评估指标
- `print_evaluation_results(results)`: 格式化输出

**评估指标**:
- 准确率 (Accuracy)
- F1分数 (F1-Score)
- 混淆矩阵 (Confusion Matrix)
- 分类报告 (Classification Report)

### 3.3 服务层 (api_service.py)

**核心职责**:
- 提供REST API服务
- 处理HTTP请求
- 模型推理

**API端点**:
- `GET /health`: 健康检查
- `GET /model_info`: 查询模型信息
- `POST /predict`: 执行预测

**请求/响应格式**:
```json
// 请求
{
  "features": [5.1, 3.5, 1.4, 0.2]
}

// 响应
{
  "prediction": 0,
  "probability": [0.9, 0.05, 0.05]
}
```

## 4. 扩展性设计

### 4.1 模型扩展
- 在`model_training.py`中添加新算法
- 在`VotingClassifier`中添加新的基分类器
- 无需修改评估和服务层代码

### 4.2 评估指标扩展
- 在`model_evaluation.py`中添加新的指标计算
- 修改`evaluate_model`函数
- 无需修改训练和服务层代码

### 4.3 API接口扩展
- 在`api_service.py`中添加新的路由
- 保持现有接口兼容

## 5. 部署架构

### 开发环境
- 单节点部署
- 使用Flask内置开发服务器

### 生产环境建议
- 使用Gunicorn作为WSGI服务器
- 配置Nginx反向代理
- 考虑容器化部署（Docker）

## 6. 性能考量

### 训练性能
- 数据量小时：一次性加载所有数据
- 数据量大时：考虑分批处理（当前实现暂不支持）

### 推理性能
- 模型预加载（服务启动时加载）
- 单请求延迟：毫秒级
- 并发支持：取决于WSGI服务器配置
